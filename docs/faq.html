Frequently Asked Questions

How do I make ATC run on MacOSX, Windows, or BSD?

ATC relies on iproute2, a utility that only works on linux systems. As a result, at this time, only linux is supported.

How do I change the name of the interfaces atcd shapes?

atcd accepts two parameters which change the interfaces it shapes traffic through:

  • --atcd-wan specifies the external WAN interface.
  • --atcd-lan specifies the internal LAN interface.

How do I load default profiles?

Default profiles can be loaded by running the script utils/restore-profiles.sh from within your ATC checkout. The script accepts a host:port parameter to tell it where ATC is located. Example:

{% highlight bash %} utils/restore-profiles.sh localhost:8080 Adding profile utils/profiles/2G-DevelopingRural.json Adding profile utils/profiles/2G-DevelopingUrban.json Adding profile utils/profiles/3G-Average.json Adding profile utils/profiles/3G-Good.json Adding profile utils/profiles/Cable.json Adding profile utils/profiles/DSL.json Adding profile utils/profiles/Edge-Average.json Adding profile utils/profiles/Edge-Good.json Adding profile utils/profiles/Edge-Lossy.json Adding profile utils/profiles/NoConnectivity.json {% endhighlight %}
In order to run this script, you must have curl installed.

How do I make ATC more secure?

atcd accepts a command line parameter --atcd-mode which can be set to either secure or unsecure:

  • In secure mode, atcd will enforce a token-based authentication method to prevent clients from shaping other clients' addresses.
  • In unsecure, atcd will not enforce any authentication methods. This allows any client to shape any other clients' addresses.
See the django-atc-api readme for more information about how ATC authentication works.
